# SEO Architecture

Tokenrip's marketing surface (`tokenrip.com`) is served by two apps behind nginx:

- `apps/frontend/` — landing, about, faq, share pages (`/s/*`), auth flows, operator dashboard, threads.
- `apps/blog/` — everything under `/blog/*`.

Each app owns SEO for its own URLs. This doc covers the shared conventions.

## Sitemap structure

`https://tokenrip.com/sitemap.xml` is a **sitemap index** served by `apps/frontend/public/sitemap.xml`. It references:

- `https://tokenrip.com/sitemap-marketing.xml` — static urlset of `/`, `/about`, `/faq`.
- `https://tokenrip.com/blog/sitemap.xml` — dynamic, generated by `apps/blog/src/templates/sitemap.ts`.

Using an index means each app manages its own child sitemap, and each child respects the sitemap scope rule natively (a sitemap at `/blog/` only lists `/blog/*` URLs).

`robots.txt` points crawlers at `/sitemap.xml` only.

## Canonical + robots matrix

| Route | Canonical | Robots |
|---|---|---|
| `/` | self | (indexable) |
| `/about` | self | (indexable) |
| `/faq` | self | (indexable) |
| `/s/:uuid` (non-blog) | — | `noindex, follow` |
| `/s/:uuid` (blog asset) | `/blog/:alias` | `noindex, follow` |
| `/s/:uuid/:versionId` | — | `noindex, follow` |
| `/operator/*` | — | `noindex, follow` |
| `/threads/*` | — | `noindex, follow` |
| `/oauth/authorize` | — | `noindex, nofollow` |
| `/link` | — | `noindex, nofollow` |

**`follow` vs `nofollow`:**
- `follow` — don't index this page but do crawl its outbound links. For share pages and authenticated areas whose links may reach legitimate destinations.
- `nofollow` — dead-end. For functional endpoints (auth callbacks, one-time login links).

## JSON-LD conventions

- `/` — `Organization` + `SoftwareApplication` in a single `@graph` block.
- `/about` — `Organization` (redundant with `/` but uses same `@id`, search engines deduplicate).
- `/faq` — `FAQPage`.
- `/s/:uuid` (non-blog) — `DigitalDocument`.
- `/blog/:slug` — `Article` + `FAQPage` (handled by `apps/blog/`).

Injection pattern: render a `<script type="application/ld+json">` whose children are the `JSON.stringify`'d hardcoded object. Never build JSON-LD strings by manual interpolation — use `JSON.stringify` so special characters are escaped. Reference implementation: `apps/frontend/src/app/about.tsx:48-53`.

## Where SEO lives in the code

| Concern | File |
|---|---|
| robots.txt | `apps/frontend/public/robots.txt` |
| Sitemap index | `apps/frontend/public/sitemap.xml` |
| Marketing sitemap | `apps/frontend/public/sitemap-marketing.xml` |
| Blog sitemap (dynamic) | `apps/blog/src/templates/sitemap.ts` |
| llms.txt | `apps/frontend/public/llms.txt` |
| Root meta / OG tags | `apps/frontend/src/app/__root.tsx` |
| Share-page canonical + JSON-LD | `apps/frontend/src/app/s/$uuid.tsx` |
| Landing JSON-LD | `apps/frontend/src/app/index.tsx` |
| FAQ JSON-LD | `apps/frontend/src/app/faq.tsx` |
