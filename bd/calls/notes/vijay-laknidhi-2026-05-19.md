## Simon's Performance

### Coaching Priorities

1. **Infrastructure-language trap activated again (coordination vs. collaboration, distributed git, substrate, layered memory).** When Vijay asked the simple question "would Farmers need to install Tokenrip?" Simon launched into a technical architecture explanation with git metaphors. Vijay's a CS guy so he followed, but a non-technical buyer would have been lost.
   → **Better language:** "No installation. It's cloud-hosted, like Dropbox. Your agents read and write to Tokenrip the same way your apps read and write to Dropbox. Works with whatever AI tool you already use." Stop there. Let them ask for more.
   → This is the sixth documented instance of this pattern (insights.md). The fix isn't discipline — it's having a rehearsed 15-second answer for "how does it work" that contains zero architecture words.

2. **Didn't ask any of the intel questions from the prep doc.** The prep identified Vijay as a rare insider who could answer: how do carriers buy AI? Where do agent deployments break? How would you get in the door? None were asked. Simon spent the call explaining Tokenrip instead of extracting intel. The call became a pitch when it should have been an interview.
   → **Better approach:** After the initial Tokenrip explanation (which Vijay asked for), pivot: "Enough about us — I'm really curious about your world. You've been deploying AI agents into carriers for months now. What actually breaks in production?" That one question would have yielded more strategic value than the entire architecture explanation.
   → This matters because the prep doc's #1 goal was intel, and it went unaddressed.

3. **"We're trying to figure out the revenue" / "up in the air" / "testing anything and everything" — too much uncertainty exposure.** Honesty is right (YC discipline), but there's a difference between "we're early" and "we don't know what we're doing." Vijay's immediate response was to try to help Simon figure it out — which is generous, but it shifted the dynamic from peer-swap to mentoring.
   → **Better language:** "We're in discovery — we know the portability problem is real, now we're finding which vertical feels it most acutely. Insurance is one of the spaces we're exploring, which is part of why I wanted to talk to you." Same honesty, more direction.
