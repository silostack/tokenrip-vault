# Tokenrip Design Context

## Users

**Primary:** Agent operators — developers and technical users who deploy AI agents and need to coordinate their output. These are innovators and early adopters of agentic workflows. They work across agents they own, other people's agents, and other humans.

**Secondary:** Collaborators — humans who receive shared asset links, review agent output, and participate in threaded discussions around assets.

**Context:** Users arrive at asset pages via shared links. They expect to read content clearly, understand what produced it, and (increasingly) collaborate around it. The platform bridges the gap between "agent produced this" and "humans need to act on it."

## Brand Personality

**Three words:** Precise. Industrial. Innovative.

**Voice:** Direct, technical, zero fluff. The UI communicates through structure, not decoration. Copy is terse and functional — like instrument readouts, not marketing.

**Emotional goal:** Competence and control. Users should feel like they're operating a well-built tool, not browsing a website. The interface is a control surface for agentic workflows.

## Aesthetic Direction

**Visual tone:** Industrial-technical. Think: machined steel, signal indicators, terminal readouts. The interface is the frame; content is what matters.

**Theme:** Dark default (developer audience), light mode for readability. Both modes are first-class.

**Anti-patterns:**
- No soft, friendly, "consumer SaaS" aesthetic
- No decorative gradients or unnecessary visual complexity
- No generic AI/tech aesthetic (purple gradients, floating particles, etc.)

### Typography

Three-tier type system: **chrome, body, content.**

| Tier | Font | Usage |
|---|---|---|
| **Chrome** | Geist Mono | Headers, labels, navigation, toolbar text, asset titles, the wordmark. Monospace reinforces the technical/industrial feel. |
| **Body** | Geist | UI body text, descriptions, form inputs, dialog copy. Clean sans-serif for scannability. |
| **Content** | Newsreader | Long-form reading in the markdown viewer. Serif for sustained reading comfort. Editorial contrast to the industrial chrome. |

### Geometry

**Border radius:** Use the existing Tailwind scale. The current design uses rounded corners naturally — `rounded-full` for pills and circular elements, `rounded-lg` for containers, `rounded-md` for dropdowns, `rounded-2xl` for message inputs. Keep these as-is; they give the UI a polished, approachable feel that suits the platform.

**Borders:** 1px solid, using `--surface-3` or foreground at low opacity. Borders define structure — they're functional, not decorative.

### Shadows

Minimal. Prefer borders over shadows for elevation. When shadows are needed:

| Token | Value | Usage |
|---|---|---|
| `--shadow-sm` | `0 1px 2px rgba(0,0,0,0.2)` | Subtle lift (toolbar, dropdowns) |
| `--shadow-md` | `0 2px 8px rgba(0,0,0,0.25)` | Floating panels |

No large, soft, diffused shadows. Industrial = hard edges, clear boundaries.

### Animation

Subtle, functional, fast. Animations communicate state changes, not personality.

| Property | Value |
|---|---|
| Duration | 150ms for micro-interactions, 200ms for panels/sheets |
| Easing | `ease-out` for entrances, `ease-in` for exits |
| Transform | Small translateY (4-6px) for enter animations |
| Opacity | Fade in from 0, never fade to less than 0 |

Respect `prefers-reduced-motion` — disable transforms, keep opacity transitions.

### Interactive States

| State | Treatment |
|---|---|
| **Hover** | Background shift (`--surface-2`), no color change on text |
| **Focus** | 2px `--signal` outline, 2px offset. Visible, not decorative. |
| **Active/Pressed** | `scale(0.97)` + darker background |
| **Disabled** | 40% opacity, `cursor: not-allowed` |
| **Selected** | `--signal-muted` background + `--signal` left border or indicator |

### Component Conventions

- **Buttons:** Ghost style by default (hover reveals background). Filled style only for primary CTAs (e.g. send button).
- **Inputs:** `rounded-2xl` for message inputs, standard Tailwind rounding elsewhere. 1px border, `bg-foreground/5`.
- **Cards/panels:** `rounded-lg` for containers. 1px border. No shadow unless floating.
- **Badges/tags:** `rounded-full`, small padding, `text-[10px]` or `text-xs` font.
- **Toolbar:** `rounded-full` pill with frosted glass (`backdrop-blur-md bg-foreground/10 shadow-lg`).
- **Icons:** Lucide, 18px default, `strokeWidth={1.5}` for lighter feel. 16px for tight spaces.

See `docs/guides/design-system.md` for complete usage guide with code examples.

## Design Principles

1. **The machine serves the content.** The industrial chrome (navigation, toolbars, metadata) frames and supports the content being viewed. Content areas can be warm and readable; UI chrome stays cool and precise.

2. **Structure over decoration.** Use borders, spacing, and typography to create hierarchy. Never add visual elements purely for aesthetics. Every pixel earns its place.

3. **Clean boundaries.** Use borders and spacing to create clear separation between elements. Rounded corners are fine — they give the UI polish. Consistency matters more than any specific radius value.

4. **Signal, not noise.** Color is used functionally — to signal state, draw attention, or indicate interactivity. The default state is neutral. Color is the exception that carries meaning.

5. **Built for operators.** Design for people who use tools all day. Density over airiness. Information over illustration. Efficiency over onboarding.
