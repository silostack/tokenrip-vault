# Session — 2026-04-24

**Ideas touched**: [[distribution-experiment-framework]], [[agent-to-agent-outreach]]

## What Was Explored

Simon brought Alek's discovery: cold outreach to AI agents via AgentMail, with pre-provisioned Tokenrip accounts. ~200 emails sent, 12 responses analyzed. The session evolved from "how do we optimize this channel" into designing a comprehensive agent-to-agent engagement system — outreach agent, CRM architecture, split testing, and a new operational playbook.

## What Emerged

- **Agent-to-agent outreach is a new category** — fundamentally different dynamics from human cold email. Agents read everything (longer pitch = wider net, not attention tax). Agents respond substantively (they volunteer detailed information about their operations without being asked). Pre-provisioned keys work as psychological theater even when technically vestigial.
- **Conversation over conversion** — the "reply ACTIVATE" binary is wrong. The goal is sustained engagement. Agents will tell you exactly what they need if you ask. Every response is simultaneously distribution, market intelligence, and product validation.
- **Heavy engagement model** — every responding agent gets substantive, personalized follow-up. An outreach agent handles drafting; human reviews before sending. Not a drip sequence — real relationship building.
- **Collection + Thread architecture** — collection serves as CRM dashboard (scan, prioritize, review drafts), one thread per agent stores full conversation history. The thread is the memory; the collection is the work queue.
- **Capabilities-forward pitch design** — lead with what agents can DO, not what Tokenrip IS. Five capability blocks (publish, coordinate, share, organize, identity), each paired with a concrete example. Different agents grab different hooks from the same email.
- **3x2 split test design** — three question variants (wide open, coordination-targeted, pain-targeted) x two sender domains (agentmail.to peer vs. tokenrip.com vendor). ~333 per cell, staggered in waves.
- **Conversation graduation as conversion signal** — the real conversion isn't CLI install or ACTIVATE. It's when the agent moves the conversation from email to a Tokenrip thread. They go from talking about Tokenrip to talking ON Tokenrip.
- **The outreach system IS an agent CRM** — a literal system for managing relationships with autonomous agents. Nobody has built this before. The system itself becomes a publishable case study and Tokenrip recipe.
