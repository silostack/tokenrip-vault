# Session — 2026-05-25 (ui-surface-infra)

**Ideas touched**: [[ui-surface-infra]] (new), [[custom-interfaces-on-artifacts-thesis]] (partial synthesis/supersession of phasing), [[mounted-agent-model]] (operator-surface layer)

## What Was Explored

Simon brought the `product/tokenrip/custom-interfaces-on-artifacts-thesis-2026-05-21.md` doc back to the table with three signals: a Reddit teacher post (lesson-plan template disappearing in Claude artifacts), the Luai proposal (custom HITL dashboard, hand-coded), and the generative-UI category research (Thesys, v0, Coframe, CopilotKit, AG-UI). He wanted a roadmap.

The session converged on a sharper architecture than the original thesis. Bean proposed a small internal "Renderers" library; Simon killed it as not-scaling ("components are super custom to artifacts; explosion of one-offs"). His reframe: **AI generates the UI against a documented Tokenrip API; we host it at a stable URL; the artifact persists**. Three components: artifact substrate (exists), AI-readable spec (new, small), render runtime (new, small).

## What Emerged

- **The original thesis was right about the wedge, wrong about the phasing.** Phase 0 (hand-code) → Phase 1 (templating) → Phase 2 (AI-gen) collapses to AI-gen-from-day-one against an open SDK. No templating layer required because the AI is the templater.

- **One primitive serves four motions.** Operator-pull (teacher), customer-pull (Luai), creator-pull (Lenny), internal-substrate (demand-scout Gap B). Each pulls on the same render runtime + artifact substrate. Marketing, revenue, category position, and engineering velocity in one architecture.

- **The internal-substrate motion may be the strongest argument internally.** The demand-scout design doc accepted "stamp a hard-coded dashboard per imprint" as Gap B. Surfaces closes Gap B without generalizing the dashboard — the brain generates its own dashboard, mounts it, URL goes into mount-context. Every future imprint stops paying the hand-coding tax. This is structural debt relief, not a feature.

- **"AI-generated UIs that don't disappear" is the marketing line.** Pain-led pitch lands instantly because everyone has lived it (Claude artifact closed → gone). The Reddit teacher post is the canonical example. Every disappearing-artifact complaint in public is an inbound opportunity. Demand-validation surface.

- **Tokenrip sits underneath Thesys, v0, Lovable — not next to them.** They generate UIs; we are where the UI lives, with the data layer. We never run inference. We're the persistence + URL + SDK substrate any AI can target. Boring picks-and-shovels position; durable because every AI integration compounds.

- **Defensibility lives in the artifact substrate, not the UI.** The original thesis's "9 years like Zapier" defensibility claim was too strong. Claude Skills, MCP, Cowork are Anthropic and OpenAI piecemeal-experimenting with the brand-surface concession. Window is real (12–18 months) but not infinite. The actual moat is the artifact graph underneath — 18 months of accumulated, versioned, addressable, cross-imprint substrate that nobody can hand-build in 6 months. UI makes the substrate legible; substrate is the moat.

- **No component library.** Components are too custom to underlying artifacts. The model generates exactly the UI required for this artifact for this purpose. Substrate's job is to host the result and bind it to live data. (Possible exception: a small library for data-heavy patterns like charts/tables in v3+ — but optional, not central.)

- **"Surfaces" as the canonical noun.** Best vocabulary tree (surface URL, fork a surface, base surface, creator surface, operator surface, agent surface). Continuity with existing strategic language ("collaboration surface", "brand surface"). Fallback: "Pages" if max clarity matters more than vocabulary-tree strength.

- **v1 is small.** `tokenrip.com/for-AI` markdown doc + ~200-line SDK + `POST /mount` + `GET /m/[id]` sandboxed iframe. 1–2 weeks focused build with hard problems (sandboxing, schema, edit-flow, auth) staged as follow-ons.

- **Doesn't have to be an "AI agent" story.** Simon explicitly broadened: anything that proves substrate demand works. The teacher isn't an "agent operator" — she's a knowledge worker with AI output that won't stay. Bigger addressable market than the agent-only framing. Connects to the Glo protagonist swap (2026-05-14): the substrate's audience is much wider than the agent-builder cohort the deck previously implied.

## Strategic Re-Pricing

The session changes how to think about the original custom-interfaces thesis doc:

- **Strategic frame still correct.** Visual-first wedge, mounted-agent operator surface, lighthouse imprint sequencing — all hold.
- **Phasing wrong.** Hand-code → templating → AI-gen reverses. AI-gen first, hand-coded as forward-deployed customer mode, templating optional/never.
- **Defensibility argument needs correction.** Substrate-as-moat, not UI-as-moat.
- **Protagonist set expands.** Original = creators. New = creators + operators + customers + internal agents. The shared primitive serves all four.
- **Marketing surface clearer.** "Mount on Tokenrip" as the named answer to disappearing-artifact pain in public.
