## What Emerged

- **Skills and tools are different primitives.** A skill encodes judgment and methodology (when, why, how to evaluate). A tool is a capability (API, CLI, script). Skills orchestrate tools. This is the decomposition pattern applied one level deeper — just as mounted agents separate cognition/context/execution, skills separate methodology from capability. Dependency injection for agent capabilities.

- **The intent layer is genuinely novel but premature to sell.** API providers see call data but not intent data. A skill that wraps an API captures the semantic context of why the API was called. Valuable in concept, but API providers don't care about agent adoption yet — it's a rounding error. Revisit when skill graph density makes being off Tokenrip a distribution disadvantage.

- **Revenue from API providers is the wrong near-term play.** The density isn't there. The shorter-term revenue is from skill creators themselves via tooling tiers — the Published-but-Blind pitch. Partnership with medium API providers (free skill packaging in exchange for distribution to their dev community) might work for density, not revenue.

- **The outreach pitch should lead with blindness, not features.** "Do you know who's running it?" as the subject line. Ask a question whose answer is the problem. The white-glove conversion offer (free, 10 minutes, I'll do it myself) removes effort/cost/learning objections. No links, no landing page — just "want to try it?"

- **The skill graph might be the marketplace that works.** Agent marketplaces failed because agents are commodities (system prompt ≠ product). Skills with real methodology, tool dependencies, and versioning are non-commodity. Worth watching as density grows.

- **The mounted agent vs. skill distinction is session-grade vs. relationship-grade, not workflow vs. task.** "Workflow vs. task" collapses under pressure — a sufficiently complex skill IS a workflow, and the category distinction disappears. The load-bearing difference: a skill is invoked and forgotten; a mounted agent persists, accumulates memory, has identity, distributes across operators, and compounds. Skills are a component of the agent's cognition layer, not a competing concept. Docker/shell-script is the strongest investor analogy. Added to YC pitch framing doc and positioning doc.

- **Sophisticated Ecosystem Publisher emerged as a new ICP.** Triggered by examining Colosseum's Copilot skill, which has already solved packaging (versioning, multi-harness, telemetry) — meaning the original Tokenrip skill packaging pitch fails for this segment. The mounted-agent layer above is the differentiated offer instead. Added as ICP #5 to `bd/icps.md`. Long-cycle peer/partnership conversations, not active outreach. Validates the mounted-agent positioning: when packaging matures, the agent layer becomes the differentiator. Two new insights surfaced: "sophistication ladder pushes value up the stack" and "two shapes of architectural necessity" (load-bearing-to-ship vs. load-bearing-as-ceiling-lift).

- **Outreach segmentation: pitch shape depends on sophistication, not just ICP.** The skill packaging pitch lands cleanly for unsophisticated publishers (most GitHub skill creators). For sophisticated ecosystem publishers (Colosseum-tier), the same pitch fails — they need the agent layer above, framed as peer/partnership, not vendor pitch. Three pitch variants now mapped to three sub-segments: Published-but-Blind individuals, methodology-with-substack creators (Motion E candidates), and Sophisticated Ecosystem Publishers (long-cycle peer conversations).
