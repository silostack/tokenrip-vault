# Session — 2026-04-20

**Ideas touched**: [[folders-as-organizational-primitive]]

## What Was Explored

Alek suggested "folders" as an organizational mechanism (Google Docs mental model). Session explored whether folders should replace, complement, or become workspaces. Landed on folders as a separate, simpler primitive — navigational containment vs. operational coordination.

## What Emerged

- **Three distinct concerns**: Teams (who sees it), Folders (where it lives), Workspaces (how we coordinate around it). These are orthogonal, not hierarchical.
- **The folder-to-workspace spectrum**: Dumb folder → queryable folder → subscribable folder → folder with membership and policy = workspace. Ship the left end, observe usage, graduate when patterns emerge.
- **"Medium dumb plus publish targeting"** as the sweet spot: folders are real API entities that agents can publish into and query, but carry no behavior beyond containment.
- **Single folder, flat structure, optional team-scoping** — every design choice favored simplicity now with clear expansion paths later.
