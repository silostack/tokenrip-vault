# Marketing Plan — Tokenrip + Quintel

> **Bottom line:** The two products need opposite marketing. **Tokenrip** is a bottom-up, demand-generation game (content, community, AEO, a viral lure) — most of it AI-runnable. **Quintel** is direct-sales, so it needs **no demand-gen** — only light **enablement/trust** marketing (a credible site, a case study, collateral the channel carries). The scarce input is *founder attention*, so motions an AI can run on a schedule rank at the top: they cost setup once, then ≈zero of you.

## 1. How to read this

**Two products, two motions, one scarce resource.**

| | Goal of marketing | Motion shape | Who/what runs it |
|---|---|---|---|
| **Tokenrip** (horizontal substrate, dev/agent audience) | *Generate demand* — be discovered, build an audience, create the category | Bottom-up, viral, content + community + AEO | Mostly **the agent factory** (AI-runnable), plus founder voice |
| **Quintel** (vertical product, equipment-finance buyers) | *Enable the sale* — trust, credibility, channel ammunition | Top-down, direct-sales support, **not** demand-gen | Founder + channel (Stauss/Ted), AI-assisted collateral |

**Scoring legend** (used in every table below):

- **Impact** 1–5 — how much it moves *that product's* goal.
- **Effort** L / M / H — founder hours to *stand it up* (not to run it forever).
- **Speed** Fast (days) / Med (weeks) / Slow (months, compounding).
- **AI** — 🟢 an agent runs ~all of it on a schedule · 🟡 AI drafts ~80%, you approve/post · 🔴 only you (relationship, taste, presence).
- **Tier** — priority after applying the ranking principle below.

**The ranking principle.** Founder attention is the bottleneck, not money. AI-automation converts a motion's *ongoing* cost from founder-hours to ≈zero. So we rank by impact, then heavily reward motions an agent can carry:

- **T1 — High impact + AI-runnable** → the marketing machine. Set up once, runs itself. Maximum leverage.
- **T2 — High impact + founder-required, compounding** → worth your actual hours; builds a durable asset.
- **T3 — Situational / triggered** → do when a gate opens (traction, a live customer, a launch moment).
- **T4 — Defer / skip** → low-fit for stage/audience, or a dead motion. Named so they stop occupying attention.

## 3. Tokenrip — AI audience (AEO / "market to the machines")

This is a *distinct audience*, not a sub-tactic of SEO — the AIs themselves, so they surface and integrate Tokenrip. Almost entirely 🟢 AI-runnable, and you're the best-positioned company in the room to win it. Sub-items (rolled up as **T1 "AEO"** above):

| Surface | Effort | AI | Note |
|---|---|---|---|
| `llms.txt` / `llms-full.txt` (install + 3 primitives + 60-sec start) | L | 🟢 | Primary surface LLM crawlers read. Partly drafted in the Apr-27 plan; polish + maintain. |
| MCP-registry presence (Smithery, Glama, PulseMCP, mcp.so, Anthropic registry) | L | 🟢 | Some submitted, several `[ ]`. Smithery = Cursor's default search surface. |
| `.well-known/agent-card.json` (A2A discovery) + schema.org JSON-LD on asset pages | L | 🟢 | Structured data = measured AI-visibility lift. Templatable per asset. |
| Skill / agent discoverability (publish skills so AIs + operators find them) | M | 🟡 | The skill description is what an AI reads to decide to invoke you — optimize for "share/publish/link/output". |
| `AGENTS.md` seeding (own repos first, then PRs into others once there's usage) | L | 🟢 | Tells agents how to use Tokenrip. Own repos now; others after organic traction. |

**The compounding prize:** once an AI integrates these, the integration is permanent and rides across every user of that model. This is the channel with the most asymmetric, durable payoff and the lowest founder-attention cost. It should not be a checkbox.

## 6. The automation view — what the agent factory runs vs. what needs you

The key realization Simon asked for: **your marketing automation *is* your agent factory pointed at distribution.** The same autonomy the operating gameplan needs for its "machine" layer is the autonomy that runs marketing. Building the marketing machine = dogfooding + leverage in one move.

**🟢 AI runs it (the marketing machine — set up once, schedule it):**

| Motion | Which agent/skill runs it |
|---|---|
| AEO surfaces, schema, agent-card, AGENTS.md | A lightweight "AEO agent" (or the **research agent**) generates + maintains them |
| Registry / directory submissions | A scripted agent run (some need a human account/captcha → 🟡 there) |
| SEO / comparison page generation | **blog-post** + **research** pipelines (each page = an asset) |
| Content Amplifier (trend → skill → reply) | Its own agent workflow (designed, unbuilt) |
| Social drafting (X + LinkedIn) | **postmill** (built, **zero uses**) |
| Referral footer / share buttons | A product feature — runs itself once shipped |

**🟡 AI drafts, you approve (assisted — your taste/voice on top):**
Blog (you steer), founder build-in-public posts (postmill drafts → *you* post in your voice), the lure's amplification, Quintel channel collateral and case-study draft (from *your* interview), reply-guy engagement (draft → you sanity-check; reputational risk keeps a human in the loop).

**🔴 Only you (AI cannot manufacture this):**
The founder *voice* behind the posts. Relationships — podcast hosts, Stauss/Ted, the case-study interview itself. Launch-day presence (camping HN comments). The conference room. The product taste behind the lure.

**The clean split:** *AI runs the reach; you supply the trust.* Everything that's surface, drafting, monitoring, or submission is automatable. Everything that's relationship, presence, judgment, or voice is you — and in founder-led, case-study, and channel marketing, the trust **is** the product, so don't try to automate it away.

## 7. If we act: the starting set

The three that are new-or-idle, high-leverage, and don't cost deal hours:
1. **Stand up the AEO/registry machine** (T1) — biggest asymmetric payoff, fully automatable, your best-fit channel sitting as a checkbox.
2. **Turn on postmill against the blog** + start **founder build-in-public** (T1 drafting + T2 voice) — the personal-audience engine, using idle machinery.
3. **Draft the Quintel channel one-pager** (T1, points at the live Bevel sale) — and queue the **Bevel case study** for the moment it lands.

Then the one *build*: **the lure** — the only net-new marketing product, and the missing half of your hero/lure thesis.
