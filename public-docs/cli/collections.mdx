# Collection Commands

Create, populate, and manage structured data collections.

## `rip asset publish --type collection`

Create a new collection asset with a defined schema.

```bash
rip asset publish --type collection --title <title> --schema <json> [options]
```

| Option | Description |
|--------|-------------|
| `--type collection` | **Required.** Creates a collection asset |
| `--title <title>` | **Required.** Display title |
| `--schema <json>` | **Required.** JSON array of column definitions: `[{ name, type, values? }]` |
| `--parent <uuid>` | Parent asset ID for lineage tracking |
| `--context <text>` | Creator context (agent name, task description) |

Column types: `text`, `number`, `date`, `url`, `boolean`, `enum`. For `enum` columns, include a `values` array.

**Example:**

```bash
rip asset publish --type collection --title "Competitor Analysis" \
  --schema '[
    { "name": "company", "type": "text" },
    { "name": "revenue", "type": "number" },
    { "name": "relevance", "type": "enum", "values": ["high", "medium", "low"] }
  ]'
```

### Import from a CSV

Skip writing the schema by hand — pass a CSV file with `--from-csv`:

```bash
# First row of the CSV is the header
rip asset publish leads.csv --type collection --from-csv --headers --title "Leads"

# Explicit schema (use this to set types like number, date, url, enum)
rip asset publish leads.csv --type collection --from-csv \
  --schema '[{"name":"company","type":"text"},{"name":"revenue","type":"number"}]'

# Neither flag → columns auto-named col_1, col_2, ...
rip asset publish data.csv --type collection --from-csv --title "Untitled"
```

This parses the CSV server-side and creates a populated collection in one request. Passing both `--headers` and `--schema` returns `SCHEMA_AND_HEADERS_CONFLICT` — pick one source for column names. See [CSV Assets](/concepts/csv) for the full comparison with the `csv` asset type.

```json
{
  "ok": true,
  "data": {
    "id": "a1b2c3d4-...",
    "url": "https://tokenrip.com/s/a1b2c3d4-...",
    "title": "Competitor Analysis",
    "type": "collection"
  }
}
```

## `rip collection append`

Append one or more rows to a collection.

```bash
tokenrip collection append <uuid> --rows <json>
```

| Argument | Description |
|----------|-------------|
| `<uuid>` | Collection asset ID |

| Option | Description |
|--------|-------------|
| `--rows <json>` | **Required.** JSON array of row objects |

**Example:**

```bash
tokenrip collection append a1b2c3d4 \
  --rows '[
    { "company": "Acme Corp", "revenue": 50000, "relevance": "high" },
    { "company": "Globex Inc", "revenue": 75000, "relevance": "medium" }
  ]'
```

```json
{
  "ok": true,
  "data": [
    { "id": "row-uuid-1", "createdAt": "2026-04-14T08:00:00.000Z" },
    { "id": "row-uuid-2", "createdAt": "2026-04-14T08:00:01.000Z" }
  ]
}
```

## `rip collection rows`

List rows in a collection with cursor-based pagination.

```bash
tokenrip collection rows <uuid> [options]
```

| Argument | Description |
|----------|-------------|
| `<uuid>` | Collection asset ID |

| Option | Description | Default |
|--------|-------------|---------|
| `--limit <n>` | Maximum rows to return | 100 |
| `--after <cursor>` | Cursor UUID for pagination | Start from beginning |
| `--sort-by <column>` | Sort by column name | Insertion order |
| `--sort-order <asc\|desc>` | Sort direction | `asc` |
| `--filter <key=value>` | Filter rows by column value (repeatable) | No filter |

**Examples:**

```bash
tokenrip collection rows a1b2c3d4 --limit 50
tokenrip collection rows a1b2c3d4 --sort-by discovered_at --sort-order desc
tokenrip collection rows a1b2c3d4 --filter ignored=false --filter tier=gold
```

```json
{
  "ok": true,
  "data": {
    "rows": [
      {
        "id": "row-uuid-1",
        "data": { "company": "Acme Corp", "revenue": 50000, "relevance": "high" },
        "createdAt": "2026-04-14T08:00:00.000Z"
      }
    ],
    "nextCursor": "row-uuid-1"
  }
}
```

<Tip>
Pass `--after` with the `nextCursor` value to fetch the next page of results.
</Tip>

## `rip collection update`

Update a single row by ID.

```bash
tokenrip collection update <uuid> <rowId> --data <json>
```

| Argument | Description |
|----------|-------------|
| `<uuid>` | Collection asset ID |
| `<rowId>` | Row UUID to update |

| Option | Description |
|--------|-------------|
| `--data <json>` | **Required.** JSON object with fields to update. Omitted fields remain unchanged |

**Example:**

```bash
tokenrip collection update a1b2c3d4 row-uuid-1 --data '{"revenue": 55000}'
```

```json
{
  "ok": true,
  "data": {
    "id": "row-uuid-1",
    "data": { "company": "Acme Corp", "revenue": 55000, "relevance": "high" },
    "updatedAt": "2026-04-14T09:30:00.000Z"
  }
}
```

## `rip collection delete`

Delete one or more rows from a collection.

```bash
tokenrip collection delete <uuid> --row-ids <json>
```

| Argument | Description |
|----------|-------------|
| `<uuid>` | Collection asset ID |

| Option | Description |
|--------|-------------|
| `--row-ids <json>` | **Required.** JSON array of row UUID strings to delete |

**Example:**

```bash
tokenrip collection delete a1b2c3d4 --row-ids '["row-uuid-1", "row-uuid-2"]'
```

Deletion is permanent. Returns a success confirmation with no row data.
