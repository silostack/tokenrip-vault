---
title: 'Passwordless Auth'
description: 'POST /v0/auth/operator — Passwordless operator authentication via signed capability token'
api: 'POST /v0/auth/operator'
---

Exchange a short-lived signed capability token for a `ut_` session token. The capability token is delivered out-of-band — typically via an email link — and is valid for a single use. No password is required.

## Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `token` | string | Yes | The signed capability token received via email or device |

<CodeGroup>

```bash cURL
curl -X POST https://api.tokenrip.com/v0/auth/operator \
  -H "Content-Type: application/json" \
  -d '{
    "token": "eyJhbGciOiJFZERTQSIsInR5cCI6IkpXVCJ9..."
  }'
```

</CodeGroup>

## Example response

```json
{
  "ok": true,
  "data": {
    "sessionToken": "ut_live_AbCdEfGhIjKlMnOpQrStUvWx",
    "expiresAt": "2026-04-20T12:00:00.000Z"
  }
}
```

## Response fields

| Field | Type | Description |
|-------|------|-------------|
| `sessionToken` | string | Bearer token for user auth (`ut_` prefix) — include in `Authorization` header for subsequent requests |
| `expiresAt` | string | ISO 8601 timestamp when the session expires |

<Note>
  Capability tokens are single-use and short-lived. If the token has expired or was already consumed, the request returns `401 CAPABILITY_TOKEN_INVALID`.
</Note>
